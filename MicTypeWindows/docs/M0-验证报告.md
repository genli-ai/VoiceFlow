## 2. Qwen3-ASR Windows 推理查证

| 来源 | 链接 | 结论 |
| --- | --- | --- |
| 官方仓库 | https://github.com/QwenLM/Qwen3-ASR | 官方发布 0.6B/1.7B，支持 30 种语言 + 22 种中文方言；仓库 Quickstart 以 Python/vLLM/DashScope/Docker 为主，未看到官方 Windows C# / ONNX Runtime 桌面嵌入路径。 |
| HuggingFace / 社区 | https://huggingface.co/andrewleech/qwen3-asr-0.6b-onnx / https://github.com/andrewleech/qwen3-asr-onnx | 存在社区 ONNX export（FP32 + int4 MatMulNBits），理论上可走 ONNX Runtime；但不是官方交付，需在 M0 真机验证延迟、内存、打包体积和 C# 调用链。 |
| sherpa-onnx 支持列表 | https://k2-fsa.github.io/sherpa/onnx/sense-voice/index.html | sherpa-onnx 官方文档列出 SenseVoice 支持 Windows x64/x86/arm64 与 C# API；未看到 Qwen3-ASR 被 sherpa-onnx 官方列为可用模型。 |

结论：Qwen3-ASR Windows runtime **存在社区 ONNX 路线，但暂无官方桌面 C# 路线**。M0 可把它列为 C 备选，但不能直接作为 M1 默认引擎；优先仍是 sherpa-onnx + SenseVoice 或 whisper.cpp/Whisper.net。

## 7. 不依赖真机的集成路线调研

| 路线 | 资料 | Windows / C# 可用性 | 热词机制 | 初步判断 |
| --- | --- | --- | --- | --- |
| sherpa-onnx + SenseVoice | https://k2-fsa.github.io/sherpa/onnx/sense-voice/index.html / https://k2-fsa.github.io/sherpa/onnx/csharp-api/index.html | 文档明确 SenseVoice 可用于 Windows x64/x86/arm64；C# API 同时支持 streaming 和 non-streaming；NuGet 包 `org.k2fsa.sherpa.onnx` 有预编译库。 | sherpa-onnx hotwords 文档说明仅 transducer 模型支持 hotwords，SenseVoice 不是 transducer 路线，不能假设 hotwords 文件生效；MicType 的 `wrong=right` 硬替换必须保留。 | 仍是 A 首选：Windows/C# 集成成熟、CPU 友好；M0 重点测中文/中英混说和无热词时专名效果。 |
| whisper.cpp / Whisper.net large-v3-turbo | https://github.com/ggml-org/whisper.cpp / https://www.nuget.org/packages/Whisper.net/ / https://huggingface.co/openai/whisper-large-v3-turbo | whisper.cpp 明确支持 Windows MSVC/MinGW；Whisper.net 提供 .NET 包和多 runtime，CPU runtime 支持 Windows 11+。 | Whisper 本身没有等价 Qwen context hotwords；主要依赖后处理和 LLM polish，专名可用 `wrong=right` 硬替换补。 | B 稳妥兜底：集成风险低，但中文专有名词和无独显延迟需要实测。 |
| Qwen3-ASR ONNX community | https://github.com/QwenLM/Qwen3-ASR / https://huggingface.co/andrewleech/qwen3-asr-0.6b-onnx | 社区 ONNX 模型声称可用 ONNX Runtime；需要自写 C# pipeline 或包装现有 transcribe-rs/脚本。 | 官方 Qwen prompt/context 能力在 ONNX 路线是否可等价暴露未知；先不承诺。 | C 有质量吸引力，但工程风险最高；M0 只做可行性验证。 |
