### 第一轮真机冒烟记录（朋友 PC，2026-06-11，二轮骨架版含 Placeholder 引擎）

- ✓ 程序启动、托盘图标与菜单正常（中文渲染正确）
- ✓ Right Ctrl 钩子有响应（触发了录音流程与提示音）
- ✓ 录音真实启动：Windows 麦克风设置页显示 MicType「当前正在使用」；桌面应用麦克风权限开启
- ✗ **悬浮窗全程不可见**——结合上面已知风险项判定为高 DPI 偏位（`Screen.WorkingArea` 物理像素被当 DIP 用，悬浮窗定位到屏幕外）；用户把开始/错误提示音统一感知为「报错声」
- 未能有效执行：Esc 吞键、多应用、全屏、多显示器、设置窗口各项（悬浮窗不可见导致无法判断状态）——待 DPI 修复后随带识别引擎的版本一并复测
- 机器信息：Administrator 账户桌面运行；屏幕缩放比例待确认（疑似 >100%）

### 第二轮真机记录（朋友 PC，2026-06-12，M1+第三轮版，含日志回传）

机器：Windows 10 (NT 10.0.19045)，2560×1440 @ **125% 缩放**（坐实第一轮 DPI 假说）。

- ✓ **DPI 修复生效**：悬浮窗可见，日志坐标 targetPx=(955,1235) 在工作区内、targetDip 换算正确
- ✓ **SenseVoice 全链路打通**：模型下载（38min，GitHub 直连）→ 解压安装 → 加载 716-1069ms → 转写正确（3 条中文全对，预载后 46-101ms）
- ✓ 静音门：peak=0.0092 < 0.012 正确拦截
- ✓ 热键 tap、Esc 取消、explorer/notepad/微信 三应用录音均正常触发
- ✗ **新 P0 bug：投递（粘贴）环节 UI 线程死锁**。证据链：日志「Transcription completed」后无任何投递日志；转写文本已进「最近记录」（HistoryStore.Add 在 Insert 之前）；悬浮窗永久停留「识别中…」（更新代码在 Insert 之后）；**右 Ctrl 随即彻底失效**——键盘钩子回调用同步 `Dispatcher.Invoke`，傻等死锁的 UI 线程 → 钩子超时被 Windows 卸载；重启进程恢复。修复见第四轮任务书（投递超时+STA 隔离+钩子改 BeginInvoke+投递日志）
- 未测：指令模式 / UIA 选区读取（朋友未配 API Key），留待后续

### 第三轮真机记录（朋友 PC，2026-06-12，第四轮死锁修复版）

- ✓ **死锁修复全部生效**：唤醒→识别→提示「已输入」全流程顺畅，连续使用不卡死、热键不失效；投递日志链完整（Deliver enter → Insert begin → clipboard write → SendInput → result）
- ✗ **新发现（已修复）：粘贴按键被系统整批拒收**——日志铁证 `SendInput key=0x56 sent=0/4`：INPUT 结构体在 x64 上只有 32 字节（union 缺 MOUSEINPUT），系统要求 40 字节，所有 SendInput 调用返回 0 但代码仍报 Pasted。修复（main @ 9a67d8f，CI 绿）：补全 union、校验 SendInput 返回值、拒收时降级为「文本留在剪贴板 + 提示 Ctrl+V」且不恢复旧剪贴板。待第四轮真机复测打字真实出现

### 第二位测试者记录（Win11 22631 双屏，2026-06-12）

- 现象：「点保存设置没反应」。日志链：两次启动日志间隔 10 秒且无退出（**双实例并行**，程序无单实例锁）→ settings.json 被互踩写坏 → 此后每次启动 `Failed to load settings`（AppLanguage 解析失败）→ 静默回退默认配置（用户选的右 Shift 丢失）
- 另：保存成功提示写在 tab 内部小字，按钮在窗口右下角，视觉上关联不上
- 修复（main @ 5e97368）：单实例 Mutex（二次启动提示后退出）、Save 原子写入（tmp+move）、宽容枚举反序列化（坏字段回退默认值而不是毁掉整份设置）、坏文件备份为 settings.corrupt.json 后自动重建、「已保存 ✓」反馈移到保存按钮旁且异常可见
- 多屏数据（竖屏副屏 + 横屏主屏，scale=1.0）：悬浮窗坐标计算正确
